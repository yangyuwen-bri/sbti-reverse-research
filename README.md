# SBTI Reverse Research

## 在线查看

**GitHub Pages：**
[https://yangyuwen-bri.github.io/sbti-reverse-research/](https://yangyuwen-bri.github.io/sbti-reverse-research/)

> 一个基于公开页面信息的独立研究项目，聚焦两个问题：
> `SBTI` 的人格结果是如何被算法算出来的，以及不同人格在理论作答空间里各占多少。
>
> ⚠️ 本仓库不是官方实现，也不是心理诊断工具；内容仅用于研究、解释、可视化展示和产品讨论。

---

## 这个研究在做什么

这个项目不是简单复述测试结果，而是把一套“看起来像玄学”的人格测试，拆成可以验证的结构：

- 题目如何被聚合成 15 个维度
- 每个维度如何从原始作答落到 `L / M / H`
- 人格模板如何与 15 维向量做匹配
- 为什么有些人格天然更容易被命中，有些人格更稀有
- `DRUNK` 这样的特殊分支会如何改变整体分布

换句话说，这个仓库想回答的不是“你是什么人格”，而是：

- 这个测试背后的算法到底是什么
- 这套人格分类是不是均匀分布
- 哪些人格是“大类”，哪些人格是“小类”
- 某一个人格结果，究竟是题目结构导致的，还是模板覆盖范围导致的

---

## 研究价值

这个项目最值得看的部分，我觉得主要有三块：

### 1. 算法拆解

把原本隐藏在页面脚本里的逻辑，整理成可以阅读、讨论和复现的规则：

- 从作答到 15 维分档的转换规则
- 从 15 维向量到人格模板的匹配规则
- 相似度、距离、阈值和分支逻辑的具体含义

这部分的价值在于，它把“测试结果”变成了“可解释的计算过程”。

### 2. 人格占比统计

这个项目不只关心单个样本会落到哪里，还关心整个理论作答空间的全局分布：

- 所有人格是否等概率出现
- 哪些人格覆盖的答案组合更多
- 哪些人格在模型里天然更常见
- 排除特殊分支后，正常人格之间的占比会如何变化

这部分的价值在于，它把“人格标签”从叙事层面拉回到了分布层面。
“哪个人格最多、哪个人格最少、热门人格是不是因为模板覆盖更大”这些问题，只有把分布真正算出来才有答案。

### 3. 可视化展示

除了脚本和文档，仓库还提供一个可以直接部署到 GitHub Pages 的展示页，用来直观看这些结论：

- 人格占比分布
- 模板之间的差异
- Top1 / Top2 的边界解释
- 单维度微调后的命中变化

这部分的价值在于，它让研究结果不是只停留在代码里，而是能被直接阅读、转发和讨论。

## 核心发现

在当前公开页面逻辑下，本项目的主要发现是：

1. 这不是随机抽人格，而是确定性映射。
   先按题目得分得到 15 维 `L/M/H`，再与人格模板做最近邻匹配。

2. 不同人格的理论占比并不均等。
   在“答案组合等概率”这个假设下，不同人格承接的组合数差异明显。

3. 特殊分支 `DRUNK` 会稳定吃掉一部分总概率质量。
   这意味着整体分布和“排除特殊分支后的常规人格分布”不是一回事。

4. 人格结果不仅取决于标签名称，还取决于模板在 15 维空间中的覆盖范围。

详见 `docs/03_findings.md`。

---

## 为什么这个点值得研究

因为这类人格测试通常只展示结果，却不展示结果背后的结构。

这个项目关心的正是这些通常被隐藏起来的问题：

- 题目如何被聚合成维度
- 维度如何被映射成人格
- 不同人格是否天然就有不一样的命中概率
- “某种人格很多”到底反映的是用户特征，还是模板覆盖范围

一旦把这些问题展开，讨论的对象就不再只是人格标签本身，而是这套分类系统的实现机制和分布特征。

## 快速开始

### 1. 安装依赖

```bash
cd /Users/yuwen/sbti-reverse-research
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 2. 运行完整分析

```bash
python scripts/analyze_sbti.py
```

运行后会生成：

- `docs/data/summary.json`
- `docs/data/type_distribution_full.csv`
- `docs/data/type_distribution_non_drunk.csv`
- `docs/data/mapping_example.csv`

### 3. 生成图表

```bash
python scripts/plot_results.py
```

会输出：

- `docs/figures/full_distribution.png`
- `docs/figures/top10_full.png`
- `docs/figures/non_drunk_distribution.png`

### 4. 生成静态展示页

```bash
python scripts/generate_report.py
```

生成后，`docs/index.html` 可直接用于 GitHub Pages。

## GitHub Pages

仓库已预留 `docs/index.html` 作为静态入口页。

在 GitHub 仓库中这样设置即可：

1. 进入 `Settings`
2. 打开 `Pages`
3. 在 `Build and deployment` 中选择 `Deploy from a branch`
4. 选择 `main` 分支和 `/docs` 目录

启用后，你的页面会发布在：

`https://<你的用户名>.github.io/<仓库名>/`
